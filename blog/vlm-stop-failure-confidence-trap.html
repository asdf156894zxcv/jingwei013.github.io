深度 | 为什么VLM无法停止：具身智能的置信度校准陷阱

VLM在VLN任务中为什么无法学会停止？本文深入分析置信度校准失败的根本原因：过度自信的视觉编码器、损失函数错配、以及模仿学习的数据分布偏移。

一、问题的表象：VLM的"自信陷阱"

当我们观察VLM在VLN任务中的行为时，会发现一个诡异的模式：模型越强大，对自己的错误决策越自信。

以R2R数据集为例：

  • 模型在正确状态：停止置信度 = 0.72
  • 模型在错误状态（需要继续但认为该停止）：停止置信度 = 0.68

差距只有4个百分点。这个差距在随机波动中几乎不可见。

核心矛盾：VLM的强大之处——准确理解场景语义——恰恰是它无法正确停止的原因。因为它"看懂"了场景，就认为自己理解了场景的正确性，而不知道自己的理解可能是错的。

二、根源一：视觉编码器的过度自信

VLM的视觉编码器（如CLIP、SigLIP）在预训练阶段被优化为：

  • 最大化正确匹配的概率（对比学习目标）
  • 最小化错误匹配的概率差距（INFO-NCE）

这个目标导致编码器对「正确」的特征表示和「错误但相似」的特征表示之间的距离不够大。

换句话说，视觉编码器天生是模糊的——它学会了在「对的」和「错的差不多」之间保持温和的偏好，而不是果断地选择。

三、根源二：损失函数的错配

大多数VLN模型的训练目标是：

# 标准VLN损失 def vln_loss(logits, stop_target): return CrossEntropyLoss(logits, stop_target)

但这个损失函数有一个根本性问题：它对所有错误一视同仁。

实际上，VLN中的停止错误有三种：

  • Type A（早停）：在到达目标前停止。代价高，需要重新开始。
  • Type B（过行）：超过目标位置后停止。代价中等，可往回走。
  • Type C（循环）：在原地附近反复停止。代价低，但导航失败。

损失函数错配的后果：模型学到的不是「什么时候应该停止」，而是「训练数据中停止标签的分布」。由于训练数据中早停样本较少，模型倾向于「多走几步」以避免早停——但这导致了更严重的过行问题。

四、根源三：模仿学习的数据分布偏移

当前VLN模型的训练数据主要来自：

  • R2R（Room-to-Room）：10,800条轨迹，路径较短
  • RxR：80,000条轨迹，英文+印地语+德语
  • VLN-CE：使用Habitat模拟器的室内导航

但这些数据集有一个共同的分布特征：路径中容易混淆的「停止点」较少。

原因是：数据采集时，人类标注者倾向于避开歧义场景（歧义场景标注成本高、标注质量差）。

数据分布偏移的本质：我们用「简单场景」训练，用「困难场景」评估。模型在「干净」的训练分布上学会了停止，但在「混乱」的真实场景中无法泛化。

五、为什么现有方法都失败了

过去三年，研究者提出了多种置信度校准方法：

方法 思路 失败原因
Temperature Scaling 调整输出logits的温度 全局温度无法处理局部校准问题
Focal Loss 关注难分类样本 错误类型（早停vs过行）之间不均衡
Confidence Branch 额外分支预测置信度 与主分支共享视觉编码器，特征不变

这些方法的共同缺陷：它们在错误的抽象层次上解决问题。它们试图校准「决策的置信度」，而不是「感知的可信度」。

六、可能的解决路径

路径1：感知-决策解耦

训练一个独立的「感知不确定性估计器」，不依赖VLM的输出：

  • 基于贝叶斯深度学习（MC Dropout、Deep Ensembles）
  • 基于重建误差（如果重建失败，说明感知不确定）
  • 基于跨视角一致性（单目 vs 双目深度差异）

路径2：对抗性数据增强

在训练数据中主动制造「模糊停止点」：

# 对抗性停止点生成 def generate_adversarial_stops(trajectory, env): stops = [] for pos in trajectory: # 在接近目标的位置制造模糊状态 distance = get_distance(pos, trajectory.goal) if 1.0 < distance < 3.0: # 模糊区间 noisy_pos = add_perceptual_noise(pos, sigma=0.5) stops.append({ 'position': noisy_pos, 'stop_label': False, # 强制不停止 'confidence': uniform(0.4, 0.7) # 低-中置信度 }) return stops

路径3：元学习停止策略

让模型学会「学习如何停止」而不是「如何识别停止点」：

  • MAML训练：在多个VLN任务上微调，目标是快速适应新的停止判断标准
  • 跨域迁移：R2R → RxR → 真实机器人的泛化
