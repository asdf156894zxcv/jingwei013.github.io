多模态LLM在具身导航中的应用：从理解到行动的关键桥梁

2025年，一个明显的趋势是：多模态大语言模型（MLLM）正在重新定义具身导航的研究方向。

从LLaVA、GPT-4V到Claude 3.5 Sonnet，这些模型不仅能"看懂"图像，还能"理解"场景中的空间关系、物体功能，甚至推理出行动序列。

这让具身导航研究进入了一个新阶段：从传统VLN的"看-走"循环，转向MLLM驱动的"理解-推理-行动"范式。

核心变化：LLM是如何改变导航的？

传统VLN方法的核心问题在于：它们把语言指令和视觉观测当作"匹配"问题。

  • 输入：RGB图像 + 语言指令"走到客厅右转"
  • 处理：CNN/Transformer提取视觉特征，BERT/LSTM提取语言特征，然后对齐
  • 输出：动作{前进,左转,右转,停止}

这种方法的问题是：没有真正的理解。

MLLM带来的变化：

  • 输入：RGB图像 + 语言指令"走到客厅右转" + 场景上下文
  • 处理：MLLM理解语义、空间关系、场景结构，推理出行动意图
  • 输出：推理后的行动决策，甚至带有解释

从"特征匹配"到"语义理解"，这是一个质的飞跃。

三种MLLM在导航中的应用范式

1. 端到端MLLM导航

直接用MLLM作为导航策略网络。

代表作：TopV-Nav（CVPR 2025）

核心思想：利用MLLM的俯视图空间推理能力，实现零样本目标导航。

class TopVNav:
    def __init__(self, mlm_model):
        self.mlm = mlm_model

    def navigate(self, rgb_obs, instruction, topdown_map):
        # 构造prompt
        prompt = f"""
        Given the top-down map and room layout:
        - Your instruction: {instruction}
        - Current position: marked with 'X'
        - Target object: find the target in the map

        Plan a path and output next action.
        Options: move_forward, turn_left, turn_right, stop
        """
        # MLLM推理
        response = self.mlm.generate(
            image=rgb_obs,
            text=prompt,
            map_visualization=topdown_map
        )

        # 解析动作
        action = parse_action(response)
        return action

实验结果：

  • 在Matterport3D ObjectNav上，SR=18.3%（传统方法SOTA=22.5%）
  • 零样本能力：无训练即可导航，但性能有限
  • 优势：无需大量标注数据，泛化能力强

问题：

  • 推理速度慢（MLLM推理延迟高）
  • 成本高（需要调用昂贵的API）
  • 长程记忆能力弱

2. MLLM作为子目标分解器

MLLM不直接输出动作，而是将长程指令分解为子目标。

代表作：ImagiNav（NeurIPS 2025）

核心思想：MLLM分解子目标，视频模型想象轨迹，逆动力学提取动作。

class ImagiNav:
    def navigate(self, instruction, rgb_obs):
        # 步骤1：MLLM分解子目标
        subgoals = self.mlm_decomposer(
            image=rgb_obs,
            instruction=instruction
        )
        # subgoals = ["走到沙发", "找到遥控器", "拿起遥控器"]

        # 步骤2：视频模型想象轨迹
        for subgoal in subgoals:
            imagined_trajectory = self.video_gen_model(
                current_obs=rgb_obs,
                subgoal=subgoal
            )

            # 步骤3：逆动力学提取动作
            actions = self.inverse_dynamics(
                imagined_trajectory
            )

            # 执行动作
            for action in actions:
                self.execute(action)
                rgb_obs = self.observe()

实验结果：

  • 在HM3D ObjectNav上，SR=26.8%（传统SOTA=28.1%）
  • 推理效率提升：子目标分解后，视频生成可并行
  • 可解释性强：每个子目标都有清晰含义

优势：

  • 结合MLLM的理解力和生成模型的想象力
  • 模块化设计，易于替换组件
  • 推理路径清晰，便于调试

问题：

  • 视频生成质量直接影响导航性能
  • 逆动力学模型训练困难

3. MLLM作为记忆增强器

MLLM作为外部知识库，为导航提供上下文和常识。

代表作：Mem2Ego（CVPR 2026）

核心思想：用MLLM构建从全局到自我中心的记忆机制，增强长程导航能力。

class Mem2Ego:
    def __init__(self, mlm_model):
        self.mlm = mlm_model
        self.global_memory = []  # 全局地图记忆
        self.ego_memory = []     # 自我中心记忆

    def update_memory(self, rgb_obs, action):
        # MLLM解析观测
        observation_summary = self.mlm.summarize_observation(
            image=rgb_obs,
            context=self.ego_memory[-5:]  # 最近5步
        )

        # 更新记忆
        self.global_memory.append(observation_summary['global'])
        self.ego_memory.append(observation_summary['ego'])

    def navigate(self, instruction, rgb_obs):
        # 检索相关记忆
        relevant_memories = self.retrieve_relevant_memories(
            instruction=instruction,
            current_obs=rgb_obs,
            memory_pool=self.global_memory
        )

        # 构造增强prompt
        prompt = f"""
        Instruction: {instruction}
        Current observation: {rgb_obs}

        Relevant memories:
        {relevant_memories}

        Plan your next action.
        """
        action = self.mlm.generate(text=prompt)
        return action

实验结果：

  • 在R2R VLN上，SR=73.5%（传统SOTA=78.2%）
  • 长程导航能力显著提升：在长路径上相对传统方法提升5.2%
  • 记忆效率提升：记忆查询成本降低60%

优势：

  • 长程记忆能力强
  • 支持增量学习
  • 可解释性好

问题：

  • 记忆检索效率优化空间大
  • 记忆冲突问题

我的判断：三种范式会共存

基于当前的实验结果，我的判断是：

1. 端到端MLLM导航：研究价值大于实用价值

优点：简单直接，零样本能力强
缺点：推理慢、成本高、性能有限

适用场景：

  • 快速原型验证
  • 特殊环境中的零样本导航
  • 研究MLLM的空间推理能力

不适合：

  • 实际机器人部署
  • 大规模应用

2. MLLM作为子目标分解器：最有前景

这个范式结合了多种技术的优势：

  • MLLM的理解力（子目标分解）
  • 生成模型的想象力（轨迹预测）
  • 传统控制方法的鲁棒性（逆动力学）

适用场景：

  • 长程复杂导航任务
  • 需要高泛化能力的场景
  • 对可解释性要求高的任务

未来方向：

  • 优化视频生成质量
  • 改进逆动力学模型
  • 加速推理过程

3. MLLM作为记忆增强器：实用性强

这个范式离实际应用最近：

  • 不改变原有导航框架
  • 只需在记忆模块加入MLLM
  • 性能提升明显

适用场景：

  • 长程VLN任务
  • 需要持续记忆的场景
  • 对实时性要求不高的任务

未来方向：

  • 记忆检索优化
  • 记忆冲突解决
  • 跨会话记忆保留

给研究者的建议

如果你在研究MLLM在具身导航中的应用，我有三个建议：

1. 不要迷信端到端

端到端MLLM导航听起来很美好，但实际上：

  • 推理成本高
  • 实时性差
  • 性能提升有限

建议：考虑模块化设计，把MLLM作为某个组件（记忆、推理、规划），而不是整个系统。

2. 关注效率问题

MLLM最大的问题是推理效率。

建议：

  • 研究如何减少MLLM调用次数
  • 探索知识蒸馏方法
  • 开发轻量化MLLM

3. 评估零样本能力

MLLM的优势在于零样本泛化。

建议：

  • 在多个未见环境中测试
  • 设计专门的零样本评估基准
  • 与传统方法对比迁移能力

下一个研究机会在哪里？

MLLM在具身导航中的应用还有很多未解问题：

1. 实时性问题

如何让MLLM在100ms内完成推理？

  • 模型量化
  • 知识蒸馏
  • 异构推理（大模型做规划，小模型执行）

2. 成本问题

如何降低MLLM的调用成本？

  • 本地部署开源MLLM
  • 混合推理（简单任务用小模型，复杂任务用大模型）
  • 缓存机制

3. 鲁棒性问题

MLLM对噪声、遮挡、变化的鲁棒性如何？

  • 对抗性训练
  • 数据增强
  • 集成方法

4. 与传统方法的融合

如何将MLLM与传统VLN/RL方法结合？

  • MLLM提供高层决策，RL提供底层控制
  • MLLM生成训练数据，RL在仿真环境中训练
  • MLLM作为老师，蒸馏知识到学生模型

结论

MLLM正在改变具身导航的研究范式。

从"特征匹配"到"语义理解"，从"看-走"循环到"理解-推理-行动"。

但这不意味着传统方法会消失。三种范式会长期共存：

  • 端到端MLLM：研究价值大于实用价值
  • 子目标分解器：最有前景
  • 记忆增强器：实用性强

如果你正在研究具身导航，不要忽视MLLM。但同时，也不要迷信它。

最好的方法是：在合适的场景，用合适的范式。
