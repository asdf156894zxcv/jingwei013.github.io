VLA已死，世界动作模型当立

Jim Fan的机器人行业"葬礼致辞"深度解读 · 2026年5月9日

事件回顾：红杉AI Ascent 2026上的两场"葬礼"

在2026年红杉AI Ascent大会上，英伟达机器人研究负责人Jim Fan用20分钟给机器人行业连开了两场"葬礼"：

  • 第一场：送别过去三年几乎统治具身智能的VLA（Vision-Language-Action）
  • 第二场：送别所有人以为还能再吃很多年的遥操作（Teleoperation）
Jim Fan的原话：「VLA已死，世界动作模型当立。遥操已死，人类的传感器数据长存。」

为什么Jim Fan宣判VLA"死刑"？

VLA的三大原罪

  • 泛化能力不足：VLA在单一任务上训练后在跨任务场景中迅速崩溃，而通用机器人需要的是"一通百通"
  • 数据效率低下：VLA需要海量任务-specific标注数据，Scaling Law在物理世界数据的边际收益递减
  • 物理世界理解浅：VLA将动作视为"语言的延续"，忽视了物理规律的内在约束

什么是"世界动作模型"（World Action Model）？

世界动作模型的核心思想：不是让模型学习"看到什么→做什么"的端到端映射，而是让模型先建立对物理世界的预测性理解（World Model），然后基于这个理解生成动作。

  • 类比：大语言模型不是预测"下一个词"，而是建立对语言的深层理解后生成文本
  • 世界动作模型：不是预测"下一个动作"，而是建立对物理世界的深层理解后生成动作序列
  • 关键差异：VLA = 反应式（Reactive），世界动作模型 = 预测式（Predictive）

为什么遥操作也"死"了？

遥操作（Teleoperation）一直是具身智能数据采集的主流方式——人类操控机器人录制动作数据，然后用这些数据训练策略。但Jim Fan指出了遥操作的根本局限：

  • 人类带宽瓶颈：人操控机器人的速度远低于真实世界操作速度，采集的数据分布与真实需求不匹配
  • 默会知识丢失：人类在长期实践中积累的身体性知识（motor memory、intuitive physics）无法通过遥操作完整传递
  • 规模上限明显：遥操作依赖专业操作员，采集规模受限于人力成本
新范式：「遥操已死，人类的传感器数据长存」——不是不要人类数据，而是用更好的采集方式（第一人称视频+传感器）蒸馏人类默会知识，而不是用遥操作"遥控"机器人

Scale AI、Micro1等公司正在实践这一转变：用iPhone（LiDAR版）绑在工人额头上录制第一人称动作视频，支付约15美元/小时，覆盖71个国家约4000名工人。这是"蒸馏人类"而非"遥控人类"。

对创业者的影响：谁会受影响？谁有机会？

角色影响应对策略
基于VLA的创业公司⚠️ 高风险：VLA基础模型正在commoditize快速转向场景数据积累，不依赖VLA架构
遥操作数据服务商⚠️ 淘汰：传统遥操作采集模式转向第一人称视频+传感器蒸馏管线
世界模型研究者✅ 机会：核心技术壁垒专注物理世界预测、sim2real迁移
垂直场景机器人✅ 机会：场景数据是护城河全力积累真实部署数据，训练场景专用World Model
警示：如果你的核心竞争力是"基于开源VLA模型做应用"，VLA commoditization后这个优势将快速消失。只有场景数据+场景World Model才能建立可持续壁垒。

技术路线图：如何转向世界动作模型

第一步：建立世界模型

  • 物理仿真引擎（Isaac Gym、MuJoCo）用于大规模世界模型预训练
  • 视频预测模型（Video Diffusion）用于视觉 imagination
  • 物体动力学建模：理解物体之间的物理交互

第二步：数据采集转型

  • 从遥操作 → 第一人称视频+传感器蒸馏
  • 从人工标注 → 自动化的世界模型一致性检验
  • 从单任务数据 → 跨任务、多场景的通用数据

第三步：动作生成

  • 基于World Model的预测性规划（而非端到端反应式控制）
  • Model Predictive Control（MPC）与Learning-based的结合
  • Long-horizon任务分解：世界模型负责"规划"，技能库负责"执行"

历史性时刻：2026年5月的范式转移

这不是一次普通的技术迭代，而是一次范式转移。类似2022-2023年NLP领域从"task-specific模型"转向"foundation model"的时刻，正在机器人领域重演。

对创业者的建议：不要在VLA上继续内卷了。如果你的团队有World Model方向的研究能力，现在是最好的入场时机。如果你更擅长工程落地，全力积累场景数据——世界动作模型需要场景数据进行后训练。
