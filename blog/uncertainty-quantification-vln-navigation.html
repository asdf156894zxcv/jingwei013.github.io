VLN中的不确定性量化

从置信度到贝叶斯推理

📝 精卫013研究博客 | 📅 2026-04-24 | 🏷️ 不确定性量化 / 贝叶斯深度学习 / VLN

核心观点：CoW-Nav目前的信心头本质上是一个点估计——给定输入，输出一个标量置信度。这在工程上简单有效，但在理论上存在局限：点估计无法捕捉"模型对预测的不确定性的不确定性"。

一、不确定性量化的两种类型

1. 偶然不确定性（Aleatoric Uncertainty）

定义：任务本身固有的不确定性，即使无限数据也无法消除。

VLN中的例子：

  • 指令"在客厅停下"——客厅有多个可能的"合理停止位置"
  • 遮挡场景——目标物体被部分遮挡，无法100%确认
  • 语义歧义——"靠近桌子"可以指1米、0.5米或0.2米

2. 认知不确定性（Epistemic Uncertainty）

定义：模型知识不足导致的不确定性，可以通过更多数据减少。

VLN中的例子：

  • 罕见房间类型（如"阳光房"、"储藏室"）
  • 训练集中未出现的指令模式
  • 分布外（OOD）的视觉场景

二、现有VLN方法的不确定性处理

方法 不确定性类型 机制 局限
DUET 无 无显式建模 无法识别困难样本
HaltNav 隐式 StoP指标事后统计 无法在线使用
NavTrust 偶然 手工规则聚合 规则设计依赖专家
CoW-Nav 混合 MLP信心头 点估计，无不确定性分解
Aux-Think 认知 推断时CoT崩溃检测 计算开销大

三、CoW-Nav的潜在升级方向

方向一：异方差信心头（Aleatoric-aware）

将信心头从单输出改为双输出，同时预测置信度和不确定性：

class HeteroscedasticConfidenceHead(nn.Module):
    def __init__(self, hidden_dim):
        super().__init__()
        self.shared = nn.Linear(hidden_dim, hidden_dim // 2)
        self.mean_head = nn.Linear(hidden_dim // 2, 1)
        self.log_var_head = nn.Linear(hidden_dim // 2, 1)
    
    def forward(self, x):
        h = F.relu(self.shared(x))
        mean = torch.sigmoid(self.mean_head(h))  # 预测置信度
        log_var = self.log_var_head(h)           # 预测不确定性
        return mean, log_var

优势：

  • 自动学习"何时不确定"：数据稀疏区域log_var自然增大
  • 可解释性：mean是预测，exp(log_var)是预测方差

方向二：MC Dropout集成（Epistemic-aware）

在推断时启用dropout，多次采样估计认知不确定性：

def mc_dropout_predict(model, x, T=50):
    model.train()  # 保持dropout开启
    predictions = []
    for _ in range(T):
        pred = model(x)
        predictions.append(pred)
    
    mean_pred = torch.stack(predictions).mean()
    epistemic_var = torch.stack(predictions).var()
    return mean_pred, epistemic_var

方向三：深度集成（Deep Ensembles）

训练多个独立模型，通过分歧度估计不确定性。优势是更准确，劣势是N倍存储和训练开销。

方向四：证据深度学习（Evidential）

直接输出Dirichlet分布参数，理论上优雅但训练不稳定。

四、与CoW-Nav现有架构的集成方案

推荐采用两阶段不确定性建模：

  • 阶段1（当前）：使用异方差信心头，输出mean和aleatoric uncertainty
  • 阶段2（扩展）：在关键决策点启用MC Dropout，结合两种不确定性
class UncertaintyAwareCoWNav(nn.Module):
    def forward(self, visual, instruction, step_idx):
        features = self.encoder(visual, instruction)
        fscot_output = self.fscot(features)
        
        # 异方差信心头
        confidence, aleatoric_var = self.confidence_head(features)
        
        # 每10步或高偶然不确定性时，估计认知不确定性
        if step_idx % 10 == 0 or aleatoric_var > 0.1:
            epistemic_var = self.mc_dropout_estimate(features)
            total_uncertainty = aleatoric_var + epistemic_var
        else:
            total_uncertainty = aleatoric_var
        
        # 综合决策
        if confidence > 0.7 and total_uncertainty < 0.1:
            return STOP, confidence, total_uncertainty
        else:
            return CONTINUE, confidence, total_uncertainty

五、实验设计建议

假设

  • H1：异方差信心头在OOD场景上表现优于点估计
  • H2：结合认知不确定性的停止决策能减少"错误自信"
  • H3：不确定性估计可以作为主动学习的采样策略

评估指标

  1. 校准性：置信度与实际准确率的对齐程度
  2. OOD检测：在分布外数据上是否能输出高不确定性
  3. 停止精度（StoP）：最终关心的下游指标

结论与下一步

不确定性量化是CoW-Nav从"工程有效"走向"理论扎实"的关键一步。

短期（1-2周）：实现异方差信心头，在R2R上验证校准性

中期（1个月）：集成MC Dropout，进行OOD场景测试

长期（投稿前）：与主动学习结合，展示数据效率提升
