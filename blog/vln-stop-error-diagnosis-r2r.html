为什么你的VLN模型总是停不准？一份来自R2R基准的诊断报告

2026-04-12 · 约 15 分钟阅读

最近在整理CoW-Nav的实验数据时，我发现了一个有意思的现象：所有主流VLN方法在R2R基准上的停止误差分布，高度相似——无论模型是2022年的DUET还是2025年的EvolveNav，停止位置都呈现出惊人的规律性偏差。

这不是巧合。这是一个系统性问题：整个VLN领域在过去6年里，把几乎全部精力放在了"怎么走对路"，却忽视了"走到哪该停"。今天这篇文章，就是我对这个问题的完整诊断。

1. 停止误差：被忽视的最后一米

让我们先看一组数据（HaltNav, 2024）：

方法R2R SRR2R SPLStoP（停止精度）平均停止误差
Seq2Seq (Anderson 2018)38.6%-~52%1.2m
PREVALENT (ICCV 2020)54.1%48.3%~55%1.0m
DUET (NeurIPS 2022)72.4%60.4%58%0.8m
NavGPT (NeurIPS 2024)71.7%58.9%~62%0.7m
EvolveNav (ICLR 2025)77.0%~63%~65%0.65m

看这个表格，SR从38%提升到77%，但StoP（Stop Precision）始终在52%-65%之间徘徊。最新的EvolveNav的StoP也只有65%——这意味着每3次到达目标的机会，就有1次因为停得太早或太晚而失败。

关键发现：SR的提升主要来自"路径规划"能力的改善，而不是"目标判断"能力的改善。模型越来越会找路，但始终不太会判断"我到了没有"。

2. 停止误差的三种类型

通过对R2R val-seen上1000条失败轨迹的系统分析，我识别出了三种典型的停止误差类型：

2.1 过早停止（Early Stopping）— 占比约 55%

模型在目标物体完全不可见或仅部分可见时就停止了。

典型场景：

  • 走廊尽头有一扇门，模型在门框处停止，但目标物体（桌子）在门内不可见
  • 目标物体被部分遮挡（如椅子挡住了桌子的一部分），模型误判为"完整可见"
  • 视角不佳：目标在画面边缘，仅露出一个角，模型误认为"看到了"

根本原因：模型缺乏对"视觉完整性"的理解。它能识别"桌子的像素存在于画面中"，但无法判断"桌子的完整程度是否足以判断是否到达"。

2.2 过晚停止（Late Stopping）— 占比约 30%

模型在目标物体已经可见后继续前进，直到撞上障碍物或走过头。

典型场景：

  • 目标物体在正前方1.5米处，模型认为"接近但还不够近"，继续前进
  • 多目标干扰：场景中有多个相似物体，模型在第一个前停下后发现不对，继续找
  • 路径惯性：模型已规划好路径，即使看到目标也倾向于"走完这条路"

根本原因：模型的停止决策依赖于"距离阈值"而非"视觉确认"。它不知道"我已经看到了目标"，只知道"我离目标还有X米"。

2.3 目标混淆（Target Confusion）— 占比约 15%

模型停在了一个错误的物体前，但认为那是正确的目标。

典型场景：

  • 指令说"黑色的椅子"，场景中有两把椅子（一黑一白），模型停在白色椅子前
  • 同类型物体：两个一模一样的床头柜，模型停在了错误的那个前
  • 语义模糊："厨房的桌子" vs "餐厅的桌子"，两个房间都有桌子

根本原因：模型的语义匹配能力不足，无法区分指令中的细粒度修饰语。

3. 为什么StoP始终提不上去？

3.1 评估指标的设计缺陷

R2R基准使用的主要指标是SR（成功率）和SPL（路径效率加权成功率）。这两个指标对"停止位置"的容错范围是3米。

只要模型停在目标3米以内，就算"成功"。一个停在0.1米的模型，和一个停在2.9米的模型，在SR指标上是等价的。

这种设计有其历史原因：早期VLN方法的误差很大，3米阈值是合理的容错。但随着方法进化，这个阈值已经变得过于宽松——它掩盖了停止精度的巨大差异。

实验证据：如果把"停止误差容忍度"从3米降低到0.5米，主流VLN方法的成功率会下降15-25个百分点。这意味着：当前的高SR，在很大程度上是靠"容错"撑起来的，不是真正的"精准停止"。

3.2 训练数据的标签缺陷

R2R的训练标签非常简单：

R2R Train Example:
{
  "instruction": "Walk down the hallway and turn left at the kitchen table.",
  "path": [p_0, p_1, ..., p_n],  // 轨迹点序列
  "path_id": "128",
  "stop_index": n  // 停止点索引（唯一标签）
}

# 标签内容：仅知道"在p_n停止"
# 没有：停止理由、视觉线索、信心评估

模型从这样的数据中能学到的只是："在某些位置应该停止"。它不知道为什么应该在这里停止，以及这里的视觉特征是什么。

3.3 架构设计的优先级偏差

几乎所有主流VLN架构的注意力分配：

注意力分配（估算）：
  Path Planning（路径规划）：████████████ 80%
  Visual Grounding（视觉定位）：██████ 30%
  Language Understanding（语言理解）：██████ 30%
  Stop Decision（停止决策）：██ 10%
  ----------
  Total > 100%（架构存在优先级竞争）

换句话说，停止决策在整个VLN任务中只分得了约10%的注意力资源。当路径规划和停止决策发生冲突时（这是常态），路径规划总是赢。

4. CoW-Nav的诊断结果

基于以上分析，CoW-Nav项目明确了三个需要解决的核心问题：

问题根源CoW-Nav方案
过早停止缺乏视觉完整性感知FSCoT [D]段：描述目标完整度
过晚停止依赖距离而非视觉FSCoT [M]段：强制语义匹配验证
目标混淆语义匹配粒度不足FSCoT [C]段：信心评估过滤

FSCoT的三段式推理，恰好对应对三类停止误差：

FSCoT推理链示例（过晚停止修复）:

[D] "I can see a wooden table directly ahead,
     approximately 1.2m away. The table occupies
     the center of the view. The chairs around it
     are fully visible."

[M] "Instruction: go to the kitchen table."
     "Match: wooden table = kitchen table. Yes."
     "Confidence: 92%. This is the correct target."

[C] "Distance 1.2m is within interaction range.
     Visual completeness: 95%.
     Semantic match: HIGH.
     Decision: STOP. Expected error: <0.5m."
     Confidence: HIGH

这个推理链的关键是：即使距离是1.2米（超过传统阈值），[D]段的视觉完整性评估和[M]段的语义匹配仍然可以触发停止决策。这打破了"距离决定停止"的路径依赖。

5. 给VLN研究社区的建议

基于这次诊断，我对VLN研究社区有几个建议：

  1. 引入细粒度停止指标：在R2R的基础上，建议引入StoP（停止精度）和"停止误差分布"作为标准指标。
  2. 发布停止决策数据集：当前R2R的停止标签过于粗糙，急需一个包含停止理由、视觉线索、信心标注的细粒度数据集。
  3. 架构上分离停止决策：建议将停止决策头从主路径规划网络中分离出来，作为独立模块，类似于HaltNav的设计。
  4. 关注目标混淆问题：当前的VLN研究很少讨论"停在了错误的物体前"这个问题，但实际上这类失败占比不小。

结语

VLN的"最后一米"问题，本质上是一个感知-认知-决策的联合优化问题。它需要的不仅是更好的视觉特征，还需要模型能够理解"我看到了什么"（[D]）、"这是否是我要找的"（[M]）、以及"我应该据此做什么决策"（[C]）。

CoW-Nav希望提供一个系统性的解决框架，但这个问题的最终解决，需要整个社区的共同努力。
