导航的GPT时刻：NavFoM与具身导航基础模型

2026-03-25 · 技术博客 · NavFoM, 基础模型, VLN, ObjectNav, 跨实体泛化
🚀
具身导航的GPT时刻是否已经到来？

一、当导航遇见基础模型

2023年，大语言模型横空出世，NLP领域迎来了"GPT时刻"——一个模型理解一切语言任务。此后的两年里，视觉基础模型出现了，代码基础模型出现了，机器人操控基础模型也出现了。

但导航呢？

很长时间以来，具身导航领域的研究范式是"一个任务、一个模型、一套训练数据"。做VLN就训VLN的模型，做ObjectNav就训ObjectNav的模型，换个机器人载体（比如从四足换到无人机），几乎要从头开始。这种碎片化的状态，阻碍了导航能力向真实世界的大规模迁移。

2025年9月，北京大学王鹤团队发表了一篇论文，试图终结这个局面——NavFoM（Embodied Navigation Foundation Model）。2026年1月，这篇论文被ICLR 2026接收。

这是具身导航领域的GPT时刻吗？我读完了论文，结论是：值得密切关注，但说"GPT时刻"还为时尚早。

二、NavFoM是什么

NavFoM是一个跨实体（Cross-Embodiment）、跨任务（Cross-Task）的导航基础模型。用大白话说：一个模型，同时处理四足机器人、无人机、轮式机器人、车辆四种形态，同时做VLN、目标搜索、目标跟踪、自动驾驶四类任务。

三、核心技术设计

3.1 统一架构：TVI Token

这是NavFoM最核心的设计创新。

不同机器人的相机配置差异巨大：四足机器人通常是第一视角，无人机可能是俯视或斜视，无人车的环视相机有四个、鱼眼畸变严重。历史上，这意味着每种配置都需要单独的视觉编码器。

NavFoM的方案是引入TVI Token（Temporal-Viewpoint Indicator Token，时序-视角指示令牌）：

  • T（Temporal）：时间信息。编码这条观测数据来自哪个时间步，帮助模型区分"刚才看到的"和"很久以前看到的"。
  • V（Viewpoint）：视角信息。编码相机类型、视角方向，让模型理解"这张图是俯视的"还是"这张图是鱼眼的"。
  • I（Instance）：实体信息。编码当前机器人的类型，让模型知道"我是一台无人机"。

这三类Token作为视觉观测的"前缀"输入给统一Transformer，模型从而可以处理来自任意实体、任意视角、任意时间步的数据，而无需为每种配置单独建模。

我的理解：这本质上是把"我是什么机器人、在什么视角、处于哪个时间点"这些元信息显式编码进Token，让模型学会在不同观测上下文之间对齐表示。类似于CoT（Chain-of-Thought）思想在空间-时间维度上的扩展。

3.2 预算感知时序采样（BATS）

导航任务需要看历史。但历史越来越长，Token数量随之爆炸。

NavFoM的解决方案是Budget-Aware Temporal Sampling（预算感知时序采样）：在固定的Token预算下，动态决定保留哪些历史观测、丢弃哪些。

这个设计对真实部署至关重要——800万样本训练出来的模型，最终要跑在真实机器人的边缘计算设备上，Token预算不可能无限。

3.3 双头输出

NavFoM有两个输出头：

  • 语言建模头：用于图像问答、视频问答等生成任务，自回归输出文本。
  • 规划头（Planning Head）：用于导航任务，直接输出运动轨迹（waypoints）。

这种设计使NavFoM成为一个真正的多任务模型，而不仅仅是"一个视觉编码器加一个任务头"的简单组合。

四、评估结果

NavFoM在以下基准上进行了零样本评估：

基准任务类型实体
VLN-CE RxR视觉语言导航轮式机器人
ObjNav HM3D-OVON物体目标导航四足机器人
VLN OpenUAV视觉语言导航无人机
Tracking EVT-Bench目标跟踪多实体
nuScenes / openScenes自动驾驶车辆

作者声称在多项任务上达到SOTA或极具竞争力的零样本性能，并进行了真实世界部署验证（模型运行在中央服务器，通过互联网与各机器人客户端通信）。

具体数值需要读完整论文，但核心亮点是：不需要针对特定任务微调，直接零样本泛化到新任务和新实体。

五、为什么这重要——从研究视角

5.1 从"专用模型"到"基础模型"的范式转变

过去导航研究的模式是：定义一个任务 → 收集一个数据集 → 训练一个模型 → 在该任务上刷榜。这个循环的局限性在于，每个模型只能在其训练任务上有效，换个任务几乎要从头来过。

NavFoM代表的方向是：收集海量多任务数据 → 训练一个统一模型 → 通过少量微调或直接零样本泛化到新任务。

这正是NLP领域从BERT/GPT-2到GPT-3走过的路。

5.2 对博士研究的启示

作为同在具身导航方向的研究者，NavFoM给我的最大启示有三点：

  • 架构设计比训练数据更重要。 TVI Token的设计让800万样本发挥出了超预期的泛化能力。如果用传统方法处理多实体多任务，可能需要上亿样本才能达到类似效果。
  • 跨实体是通往真实机器人的必经之路。 大多数学术研究只在一种仿真器（Habitat-Matterport、IsaacSim）上训练，真实部署时跨实体泛化是一道鸿沟。NavFoM直接解决了这个问题。
  • NavFoM是强基线。 如果你的VLN或ObjectNav研究需要对比SOTA方法，NavFoM现在是绕不开的基线。

5.3 和我的CoW-Nav工作的联系

有趣的是，NavFoM和CoW-Nav（我自己的工作）其实有潜在的互补性：

  • NavFoM：解决了"什么动作是对的"——强大的零样本导航决策能力。
  • CoW-Nav：解决了"为什么这个动作是对的"——Chain-of-World推理链，减少幻觉。

两者结合：一个NavFoM提供强大的动作预测能力，加上CoW-Nav的World Model反馈纠错机制，可能是一套更强的具身导航系统。

六、局限性：为什么说"GPT时刻"还为时尚早

尽管NavFoM是一个重要里程碑，我仍然认为"导航的GPT时刻"这个说法有些夸张，原因有三：

  1. 800万样本 vs GPT的规模。 GPT-3训练了约3000亿Token，GPT-4据估计超过10万亿。NavFoM的800万样本虽然已经很大，但和真正的"大力出奇迹"还有数量级的差距。
  2. 零样本不等于强泛化。 论文展示了零样本在多个基准上的竞争力，但具体数值和经过任务特定微调的SOTA方法相比，差距多大，论文中没有给出详细的数字对比。
  3. 跨实体泛化的真实性存疑。 800万样本中包含了四种实体，但每个实体的训练数据量和分布差异可能很大。真实世界机器人的场景复杂度远高于仿真，零样本到真实世界的迁移效果仍需更多验证。

七、未来展望：导航基础模型的下一步

基于NavFoM的工作，我预测以下几个方向会是2026-2027年的研究热点：

  1. 导航基础模型 + CoT（Chain-of-Thought）推理：NavFoM提供了强大的视觉-语言-动作对齐表示，加上LLM的推理能力（如CoW-Nav的Chain-of-World），是自然的后续方向。
  2. 更大规模的数据：借鉴机器人遥操作数据收集（如DROID数据集），收集千万级真实世界导航数据，是缩小与"真正的GPT时刻"差距的关键。
  3. 多模态世界模型 + 导航：NavFoM本质上是"感知-动作"的映射。一个能预测"如果我执行动作A，世界会变成什么样"的世界模型，和NavFoM的结合，可能会解锁更长horizon的规划能力。
  4. 端到端Perception-Prediction-Planning一体化：最近（2026年3月）出现的P³Nav论文提出了端到端的Perception-Prediction-Planning框架，在REVERIE、R2R-CE、RxR-CE上达到SOTA。这代表了另一个有潜力的方向——不是训练一个万能模型，而是让感知、预测、规划在一个框架内协同优化。

八、总结

NavFoM是具身导航领域的里程碑工作，代表了从"专用模型"到"基础模型"的范式转变方向。TVI Token的统一架构设计、跨实体跨任务的泛化能力、零样本部署到真实机器人的可行性，都是值得深入研究的重要贡献。

但距离真正的"导航GPT时刻"，NavFoM还有距离。数据规模需要进一步扩大，零样本到真实复杂场景的泛化效果需要更多验证，推理能力的结合（LLM + World Model + Navigation Foundation Model）也是未解决的开放问题。

这恰恰是博士生做研究的最好时机——领域刚刚起步，方向清晰可见，机会就在眼前。
