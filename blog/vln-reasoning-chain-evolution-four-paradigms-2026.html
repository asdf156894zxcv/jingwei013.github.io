一、第一代：朴素CoT — 完整但不可控

2023-2024年的VLN研究普遍采用朴素Chain-of-Thought（CoT），代表工作包括EVAPORATE和Aux-Think。

核心思路：让VLM在每一步推理时输出完整的思考链，类似人类的"自言自语"：

"我现在在走廊中段，看到左侧有一扇门，右侧是开放的房间。根据指令'去厨房'，厨房通常在开放空间的尽头，所以应该继续直行而不是左转..."

优势：推理过程透明，可调试，可解释。

致命缺陷：训练-推断不对称（Train-Test Inference Asymmetry）。Aux-Think论文明确指出：

  • 训练时：CoT teacher强制输出完整推理链
  • 推断时：模型倾向于"跳过"推理直接输出动作
  • 结果：训练时学到的推理能力在推断时崩溃

二、第二代：受限CoT — FSCoT的结构化方案

FSCoT（Fixed-Structure Chain-of-Thought）的核心洞察是：不是减少推理，而是约束推理的结构。

CoW-Nav采用FSCoT，将推理链分解为4个固定段：

推理段内容Token预算
感知段当前视觉观察 + 文本指令~128
定位段拓扑位置 + 场景识别~64
规划段下一步动作候选~64
停止段置信度评估 + 停止/继续决策~32

关键改进：训练时严格遵循4段结构，推断时同样严格遵循，完全对称，消除了第一代的崩溃问题。

三、第三代：置信度门控 — 自适应计算预算

Rule-VLN和VLLR代表了另一条路线：用置信度分数决定是否触发深度推理。

核心思路：简单场景走快速通道（低推理开销），复杂场景才触发慢速通道（高推理开销）。

这与人类的认知模式高度一致：我们不会对每个决策都深思熟虑——熟悉的任务靠直觉，陌生的任务才启动系统2。

CoW-Nav的置信度头正是第三代范式的实现：

  • 置信度分数 < 0.7 → 快速通道（单步动作）
  • 置信度分数 ≥ 0.7 → 慢速通道（FSCoT完整推理）

四、第四代：世界模型预测 — 长期规划能力

DreamNav和WorldSense尝试用世界模型预测未来状态，再决定当前动作。

核心思路：不只是"我现在在哪里"，还要预测"如果我往左走，3步后会到达哪里"。

这带来了更强的长期规划能力，但代价是推理开销指数级增长——每一步都需要模拟多条可能的未来轨迹。

五、CoW-Nav的定位：站在第二代与第三代之间

当前的CoW-Nav正处于第二代→第三代的过渡阶段：

  • FSCoT（✅ 已实现）：结构化4段推理链，训练-推断对称
  • 置信度头（🔄 进行中）：自适应深度路由，根据场景难度选择推理深度
  • 世界模型（⏳ 待探索）：长期规划能力，推理开销大

这个定位是经过深思熟虑的：

  • 不盲目追求第四代（推理开销太大，实机部署困难）
  • 但也不停留在第一代（训练-推断不对称，效果不稳定）
  • 在效率和精度之间找到平衡点

六、第五代假设：推理编译器

Tool Attention（arXiv:2604.21816）给了我一个新思路：

传统VLM推理：输入 → 完整CoT → 输出
Tool Attention：输入 → 推理IR → 编译优化 → 输出（削减97% token）

类比到CoW-Nav：

  • 当前版本：FSCoT → 固定4段推理 → 停止判断
  • 升级版本：FSCoT → 可学习推理IR → 编译优化 → 自适应停止边界

核心升级点：推理段长度由置信度动态决定，而非预定义的固定4段。

结论

VLN推理链架构的进化史，本质上是"效率与精度的博弈史"。

CoW-Nav选择FSCoT作为基座，是因为它解决了第一代的核心问题（训练-推断对称），同时为第三代（置信度门控）提供了结构化接口。

这个选择务实且有前瞻性——既能在当前硬件条件下高效部署，又为未来的世界模型集成留下了扩展空间。
