# Case 108: 小米机器人VLA——47亿参数端到端具身智能模型

## 基本信息

| 项目 | 内容 |
|------|------|
| 公司 | 小米 (Xiaomi Robotics) |
| 产品 | Xiaomi-Robotics-VLA |
| 参数量 | 47亿 (4.7B) |
| 开源时间 | 2026年2月 |
| 类型 | VLA (Vision-Language-Action) |
| 领域 | 具身智能 / 机器人操作 |

## 公司背景

小米在机器人领域的布局始于2021年CyberOne仿生机器人和CyberDog四足机器人。2025年起，小米加大具身智能投入，成立Xiaomi Robotics团队，专注于VLA架构研发。2026年2月开源Xiaomi-Robotics-VLA模型，标志着小米正式进入具身智能基础模型赛道。

## 技术分析

### 模型架构
- **视觉编码器**：基于SigLIP的视觉编码器，支持多分辨率输入
- **语言解码器**：基于Qwen2.5架构的语言模型主干
- **动作头**：扩散策略动作头，支持连续动作空间输出
- **统一训练**：视觉理解、语言推理和动作输出端到端联合训练

### 核心创新
1. **实时执行**：通过架构优化实现视觉语言理解与实时执行的平衡
2. **大规模预训练**：利用小米内部大规模机器人操作数据集预训练
3. **开源策略**：完全开源权重和训练代码，推动社区发展

### 性能表现
- 在SIMPLER基准上取得竞争力结果
- 支持桌面操作、抓取放置等多种任务
- 单卡A100可运行推理

## 竞争格局

| Xiaomi-Robotics-VLA | 4.7B | 小米 | 端到端实时执行 |

| OpenVLA | 7B | 斯坦福 | 开源先驱 |
| RT-2 | 55B | Google DeepMind | 大规模VLM基础 |

## 战略意义

### 对小米
1. **生态闭环**：VLA模型 + CyberOne/CyberDog硬件 + 米家IoT生态
2. **数据飞轮**：消费级机器人部署带来真实场景数据回流
3. **人才吸引**：开源模型吸引具身智能研究人才

### 对行业
1. **降低门槛**：4.7B参数使中小团队也能参与VLA研发
2. **架构验证**：端到端VLA在消费级硬件上的可行性验证
3. **数据标准**：推动具身智能数据格式和评测标准统一

## 挑战与风险

1. **精度差距**：与55B参数的RT-2相比，4.7B在复杂任务上存在精度差距
2. **泛化能力**：跨场景、跨物体泛化仍是VLA的普遍难题
3. **实时性瓶颈**：4.7B模型在端侧设备的实时推理仍需优化
4. **数据依赖**：模型性能高度依赖训练数据质量和多样性

## 对CoW-Nav的启示

1. **VLA作为导航骨干**：Xiaomi-Robotics-VLA的架构可作为CoW-Nav停止决策的视觉语言理解骨干
2. **参数效率**：4.7B证明了在合理参数量下可以实现端到端具身智能，不需要追求超大模型
3. **开源协作**：基于开源VLA进行导航扩展，可以避免从零训练的巨大成本

## 时间线

- **2021.08**：小米发布CyberDog四足机器人

- **2026.02**：开源Xiaomi-Robotics-VLA (4.7B)
