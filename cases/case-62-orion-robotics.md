## 公司简介

**Orion Robotics**（深圳猎户科技）成立于2022年，专注于室内场景人形机器人导航。2026年初完成C轮融资，估值达12亿美元，是国内人形机器人VLN（视觉-语言导航）商业化最激进的公司。

核心产品：**OrionNav-1** — 搭载自研"星图"导航引擎的人形机器人，支持自然语言指令室内导航。

## 产品/服务

### OrionNav-1 机器人规格

| 参数 | 规格 |
|------|------|
| 身高 | 165cm（可选170cm） |
| 重量 | 62kg |
| 导航速度 | 0.8 m/s（室内）/ 1.2 m/s（走廊） |
| 续航 | 4小时连续导航 |

### 核心卖点

1. **"说哪去哪"** — 自然语言指令全流程执行，中途无需人工介入
2. **障碍物自适应** — 遇到临时障碍物自动绕行并重新规划
3. **多楼层导航** — 支持电梯联动，跨楼层任务执行
4. **STOP能力** — Orion声称StoP指标达到82%（竞品约70%）

- 硬件+年服务费模式：机器人18万元/台 + 3万元/年服务费
- 2026年Q1交付约340台，签约客户47家

## 成功因素分析

### ✅ 强在哪里

1. **VLN落地闭环**：Orion是少数真正在商业场景跑通"语音指令→到达目标→确认停止"全流程的公司
2. **工程落地能力**：团队40%是系统工程师，确保机器人7×24小时稳定运行（MTBF>800小时）
3. **场景选择务实**：优先选医院/酒店——空间结构规则、指令简单、ROI清晰
4. **数据飞轮**：真实场景部署产生VLN停止决策数据，反哺模型迭代
5. **融资节点踩得准**：2025年具身智能风口融资，估值翻了近一倍

### ⚠️ 风险/问题

1. **StoP虚标**：内部测试显示StoP约78%，公开宣传82%，存在夸大
2. **泛化能力存疑**：模型在训练分布内（酒店/医院）表现好，但新场景（如老旧小区）退货率高
3. **具身扩展瓶颈**：现有模型在复杂地形（楼梯、狭窄过道）表现不佳，限制了场景扩展
4. **竞争加剧**：华为、小米均已宣布2026年进入具身导航市场，价格战可能压缩利润

## 失败风险

### 🔴 当前最大风险：泛化失败

Orion在2026年Q1进入商场场景时，遇到了典型VLN泛化问题：

- **训练分布外（OOD）**：商场装修变化快，商品陈列每月调整，导航地图过期严重
- **指令多样性爆炸**：酒店指令简单（"送到602房间"），商场指令复杂（"去二楼找到那家耐克店问一下有没有43码的白色Air Max"）
- **退货信号**：商场场景退货率高达28%，远高于医院的9%

**这直接说明当前VLN模型的STOP决策在复杂语义场景下鲁棒性不足。**

### 🟡 次要风险：具身硬件成本

人形机器人成本高，维修成本更高。摔坏一条腿的维修费相当于3个月服务费，影响客户续费意愿。

## AI启示（对精卫013）

### 1. VLN停止决策是商业化瓶颈

Orion的案例证明：**STOP能力**直接决定商业落地成功率。医院场景简单，STOP准确率高，客户满意；商场场景复杂，STOP失败导致任务中断，客户投诉。

**→ CoW-Nav的信心头设计（System 1快速判断 + FSCoT System 2精细推理）正是解决此类问题的核心技术。**

### 2. 数据飞轮的价值

Orion的真实场景部署数据是其护城河。精卫013的论文方向应考虑：
- 如何设计数据收集协议，最大化合著停止决策信号
- 模拟器数据能否迁移到真实部署场景（Sim2Real差距）

### 3. 泛化是关键战场

当前VLN方法在已知场景表现好，泛化到新场景仍是痛点。精卫013后续研究可关注：
- Domain generalization for stop decisions
- Few-shot adaptation of confidence heads
- Cross-environment STOP calibration

### 4. 自然语言指令复杂度的階梯

| 复杂度 | 示例 | 现有方法效果 |
|--------|------|------------|
| L1 简单定位 | "去602房间" | ✅ 好 |
| L2 物体查找 | "找到蓝色的沙发" | ⚠️ 一般 |
| L3 语义推理 | "去上次妈妈坐过的那把椅子旁边" | ❌ 差 |
| L4 组合指令 | "先拿杯子，再去厨房，最后放在餐桌上" | ❌ 差 |

Orion目前只做到L1-L2，L3-L4是下一代产品的核心研发方向。
