# Case 81: Neo Robotics - 农业机器人的垂直突围之路

> 案例编号：case-81
> 案例类型：成功案例（转型成功）
> 行业：农业机器人 / 精准农业
> 数据来源：公开报道 + 行业分析
> 完成时间：2026-04-24

---

## 一、公司简介

**Neo Robotics**是一家中国农业机器人创业公司，2023年成立于深圳。创始团队来自大疆创新（DJI）和极飞科技（XAG），核心成员有6年以上的农业无人机和地面机器人开发经验。

**关键数据**：
- 成立时间：2023年3月
- 团队规模：45人（2026年4月）
- 融资轮次：A轮
- 累计融资：2.8亿元人民币
- 商业化区域：广东、广西、海南（热带水果产区）

---

## 二、产品与服务

### 第一代产品：通用农业机器人（2023-2024，已放弃）

Neo最初定位为"果园通用机器人"，一台机器尝试覆盖：
- 巡检监控
- 精准施药
- 果实采摘
- 运输搬运

**结果**：每个功能都只能做到60-70%的可用性，客户不愿意为"什么都做但什么都做不好"的机器买单。

### 第二代产品：荔枝精准采摘机器人（2025至今，成功转型）

团队痛定思痛后做出了一个关键决策——<strong>放弃通用，只做荔枝采摘</strong>。

选择荔枝的原因：
1. **采摘窗口极短**：荔枝成熟后7-10天必须采摘，否则品质骤降
2. **人工极度短缺**：热带水果产区的采摘工越来越难招，日均工资涨到400-500元
3. **技术适配度高**：荔枝果穗结构相对规则，适合机械臂+视觉方案
4. **经济价值高**：优质荔枝（如桂味、糯米糍）批发价30-80元/斤，付费能力强

**Neo荔枝采摘机器人规格**：

| 参数 | 规格 |
|------|------|
| 底盘 | 履带式，适应果园地形 |
| 机械臂 | 6轴协作臂 + 自研柔性末端执行器 |
| 视觉系统 | RGB-D + 多光谱相机 |
| AI模型 | 专有VLM（基于开源VLA微调）+ 成熟度分类模型 |
| 采摘速度 | 约8秒/果穗（人工约3秒/果穗，但需休息） |
| 日产能 | 约2000果穗/天（≈人工的1.5倍，可24h工作） |
| 电池续航 | 6小时/次，支持热替换 |
| 整机成本 | 约12万元 |

## 三、市场策略

### 核心策略：RaaS模式（按结果收费）

Neo不卖机器人，而是按采摘量收费：

- **收费模式**：0.5元/果穗（人工成本约0.8-1.2元/果穗）
- **最低采摘量保证**：承诺日采摘量不低于1500果穗，否则不收费
- **运维全包**：Neo负责维护、升级、配件更换

### 为什么选RaaS？

1. **降低客户决策门槛**：果农不需要一次性投入12万元
2. **绑定利益**：Neo的收入与采摘效果直接挂钩，客户信任度高
3. **数据飞轮**：每台机器的采摘数据回传，持续优化模型

### 2026年扩张计划

从荔枝扩展到：
- 龙眼（技术高度相似，7月上市）
- 火龙果（夜间采摘，人工更难招）
- 澳洲坚果（高价值，采摘难度大）

## 五、成功因素分析

### 1. 勇敢放弃"通用"

这是最关键的决定。2024年初，团队面临两个选择：
- 继续做通用，试图让每个功能都达到80%
- 砍掉3个功能，只留采摘，做到95%

创始人选择了后者，把巡检、施药、运输全部外包给第三方方案，只保留采摘作为核心能力。

### 2. 场景选择精准

荔枝采摘完美符合"AI替代价值公式"：
> 替代价值 = 人工短缺度 × 任务重复度 × 经济价值 × 技术可行性

四项得分都高，是极少数"AI比人更划算"的农业场景。

### 3. RaaS模式降低信任门槛

农业客户对新技术极其保守。RaaS模式让客户"零风险"试用，是打开市场的关键。

### 4. 从DJI和XAG带来的工程能力

团队在大疆和极飞的背景，让他们在硬件工程化和量产方面比纯AI团队快1-2年。

### 5. 数据闭环

每台机器每天的采摘数据回传，包括：
- 识别准确率
- 采摘成功率
- 损伤率
- 误采率

这些数据持续优化模型，形成正反馈循环。

## 六、风险与挑战

| 风险 | 严重度 | 应对措施 |
|------|--------|---------|
| 采摘季极短（每年仅30-40天） | 高 | 扩展到更多水果品类 |
| 硬件成本仍高 | 中 | 规模化生产后预计降30% |
| 极端天气影响 | 中 | 开发雨天模式 |
| 大厂进入（大疆/极飞） | 中 | 先发优势+数据壁垒 |
| 劳动力成本下降风险 | 低 | 短期不太可能 |

## 七、对AI创业的启示

1. **垂直比通用更有商业价值**：Neo用"只做荔枝采摘"换来了95%的可靠性和付费客户。通用方案的60%可靠性在商业上等于0。

2. **场景定义形态**：Neo的机器人不是人形的，而是为果园地形和荔枝树形特别设计的。形态适配场景，而非场景适配形态。

3. **RaaS是ToB机器人的最佳商业模式**：尤其适合农业、物流等对成本敏感的行业。卖结果比卖机器更容易成交。

4. **数据飞轮是护城河**：Neo每采摘一季，数据量就翻一倍，模型就更好。后来者即使有同样的技术，也缺乏数据积累。

5. **从大厂出来的团队有工程化优势**：但要注意"大厂思维"的惯性——大厂做通用产品，创业公司做垂直产品，两种思维截然不同。

## 八、与精卫013的关联

Neo Robotics的案例对精卫013的研究方向有直接启示：

- **停止决策在农业场景中是安全问题**：采摘机器人的"停止"不只是"到达目标停下"，还包括"遇到障碍物停下"、"检测到未成熟果实跳过"等场景。这与CoW-Nav的FSCoT停止推理高度相关。
- **置信度校准在垂直场景中更有价值**：采摘机器人需要知道"我对这颗果实的成熟度判断有多确定"，不确定的应该跳过而非强采。这正是CoW-Nav置信度校准思想的应用延伸。
- **世界模型+VLM的协同**：Neo的VLM做果实识别，世界模型做采摘轨迹预测，两者协同实现精准采摘。
